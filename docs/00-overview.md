# 00 - 项目总览

## 项目目标

建设一个可复用的 AI 业务任务编排平台，支持多业务场景通过配置化 Workflow 调度 Agent、Skill、Review 节点完成业务任务，并沉淀证据、产物和执行轨迹。

第一阶段以「信评报告生成」作为业务验证场景。

## 不是要做什么

本项目第一版不是：

- 不是万能 Agent 平台
- 不是自由 Planner Agent
- 不是让 LLM 自己决定所有步骤
- 不是自动生成不可追溯来源的 AI 作文
- 不是自动交易系统
- 不是 RPA 工具

## 是要做什么

第一版要做：

- 可创建业务任务
- 可按 YAML Workflow 执行步骤
- 支持 Skill 节点
- 支持 Agent 节点
- 支持 Review 节点暂停等待人工审核
- 支持 Evidence 证据沉淀
- 支持 Artifact 产物沉淀
- 支持 Trace / Audit 全链路日志
- 支持信评报告场景输出 Markdown 报告草稿

## 核心原则

### 1. Workflow 是主干

业务流程必须由系统配置控制，而不是完全交给 LLM 自由规划。

### 2. Skill 是能力

确定性任务，例如财务指标计算、文档解析、模板渲染，不应交给 LLM。

### 3. Agent 是分析节点

Agent 负责基于上下文和证据输出结构化分析，不直接执行高风险动作。

### 4. Evidence 是可信基础

报告中的关键结论必须尽量引用 evidence_id，方便人工审核和后续溯源。

### 5. Review 是把关机制

关键节点必须支持人工审核，例如评级意见、风险结论、最终报告。

## 第一业务场景：信评报告

输入：

```json
{
  "company_name": "某某科技股份有限公司",
  "period": "2024",
  "report_type": "主体信用分析",
  "materials": []
}
```

输出：

- 公司概况
- 财务分析
- 经营分析
- 行业分析
- 风险因素
- 初步评级意见
- 反方审查意见
- 证据列表
- Markdown 报告草稿

## 未来可扩展场景

| 场景 | 输入 | 输出 |
|---|---|---|
| 信评报告 | 公司、资料、期间 | 信评报告 |
| 交易决策 | 标的、行情、持仓、策略 | 交易建议 |
| 投研报告 | 公司、行业、问题 | 投研报告 |
| 合同审查 | 合同文件、审查重点 | 风险清单和修订建议 |
| 运价分析 | 报价文件、港口、箱型 | 运价分析报告 |
| 报表逻辑分析 | C# 报表代码 | 逻辑说明和复用建议 |
